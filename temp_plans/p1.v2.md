Dựa trên quy trình (workflow) bạn mô tả, đây là kế hoạch chi tiết để xây dựng **PHP Web Service (RESTful API)**. Tôi sẽ sử dụng **Laravel Framework** (khuyên dùng vì tính bảo mật, xử lý Queue và API resource mạnh mẽ) làm nền tảng ví dụ.

---

### I. KIẾN TRÚC HỆ THỐNG & CÔNG NGHỆ

1.  **Mô hình:** Client-Server (PHP Portal đóng vai trò Server API, Browser và ASP System đóng vai trò Client).
2.  **Ngôn ngữ/Framework:** PHP 8.x (Laravel 10/11).
3.  **Database:** MySQL/MariaDB (Lưu trữ thông tin User, Hồ sơ, Log giao dịch).
4.  **Authentication:**
    *   **User (Học sinh):** Laravel Sanctum (Token based) cho session đăng nhập.
    *   **System (ASP):** API Key + IP Whitelist (Chỉ cho phép IP của mạng công ty gọi vào) hoặc JWT Token riêng.
5.  **File Storage:** Local Storage hoặc AWS S3 (Lưu ảnh bằng cấp, chứng từ thanh toán).

---

### II. THIẾT KẾ CƠ SỞ DỮ LIỆU (DATABASE SCHEMA)

Bạn cần các bảng chính sau:

1.  **`users`**:
    *   `id`, `email`, `phone`, `password`, `role` (student), `is_verified` (OTP status).
2.  **`user_otps`**:
    *   `user_id`, `otp_code`, `expired_at`.
3.  **`applications` (Hồ sơ tuyển sinh)**:
    *   `id`, `user_id`, `full_name`, `dob`, `gender`, `parent_info` (JSON), `course_id`, `status` (`new`, `pending`, `approved`, `request_info`, `rejected`).
    *   `student_code` (Mã sinh viên - Nullable, sẽ có sau khi ASP duyệt).
    *   `review_note` (Ghi chú từ ASP).
4.  **`documents`**:
    *   `id`, `application_id`, `type` (bang_cap, giay_kham_sk, payment_proof), `file_path`.
5.  **`student_academic_data` (Dữ liệu sau nhập học - Cache từ ASP)**:
    *   Để giảm tải việc gọi API realtime liên tục vào mạng nội bộ, nên có bảng này để ASP đồng bộ điểm/lịch học lên PHP định kỳ.

### III. CHI TIẾT CÁC API ENDPOINTS

Hệ thống API chia làm 3 nhóm chính (Namespace):

#### NHÓM 1: PUBLIC API (Dành cho Mobile/Web Portal - Giai đoạn 1 & 3)

**A. Xác thực (Auth)**
*   `POST /api/auth/register`: Nhập Email/SĐT -> Tạo user -> Gửi OTP.
*   `POST /api/auth/verify-otp`: Nhập OTP -> Kích hoạt tài khoản.
*   `POST /api/auth/login`: Trả về Access Token.

**B. Hồ sơ (Application)**
*   `POST /api/application/step-1`: Lưu thông tin cá nhân.
*   `POST /api/application/step-2`: Lưu thông tin phụ huynh.
*   `POST /api/application/step-3`: Chọn ngành.
*   `POST /api/application/upload`: Upload file (Multipart/form-data) -> Trả về URL ảnh.
*   `POST /api/application/submit`:
    *   Input: `mpesa_code`, `payment_proof_image`.
    *   Action: Chuyển trạng thái `status` sang `pending`. Gửi email xác nhận.

**C. Sinh viên (Student Portal - Sau khi Approved)**
*   `GET /api/student/dashboard`:
    *   Logic: Check `status` == `Approved`.
    *   Return: Thời khóa biểu, thông báo.
*   `GET /api/student/fees`: Trả về công nợ.

---

#### NHÓM 2: INTERNAL SYNC API (Dành riêng cho ASP System - Giai đoạn 2)

*Cần bảo mật cao (Middleware check API Key).*

**A. Lấy dữ liệu (Pull)**
*   `GET /api/internal/students?status=pending`:
    *   Trả về danh sách hồ sơ đang chờ.
    *   Format JSON: Bao gồm đầy đủ thông tin cá nhân + Link ảnh (để kế toán đối soát).

**B. Cập nhật trạng thái (Push)**
*   `POST /api/internal/update-status`:
    *   Input:
        ```json
        {
          "application_id": 123,
          "status": "approved", // hoặc "request_info"
          "student_code": "SV2024001", // Bắt buộc nếu approved
          "reason": "Ảnh mờ, vui lòng up lại" // Bắt buộc nếu request_info
        }
        ```
    *   Logic:
        *   Nếu **Approved**: Update DB -> Cấp mã SV -> Email chúc mừng.
        *   Nếu **Request Info**: Update DB -> Email yêu cầu sửa -> User vào lại portal sẽ thấy form mở lại để edit.

### IV. KẾ HOẠCH TRIỂN KHAI (ROADMAP)

#### Bước 1: Setup & Authentication (Tuần 1)
1.  Khởi tạo Laravel Project.
2.  Thiết lập Database Migration.
3.  Viết API Register/Login/OTP.
4.  Tích hợp Email Service (SMTP Gmail hoặc SendGrid/Amazon SES) để gửi OTP.

#### Bước 2: Module Nộp hồ sơ (Tuần 2)
1.  Xây dựng các Step-form API.
2.  Xử lý File Upload (Validate định dạng ảnh/PDF, giới hạn dung lượng).
3.  Xử lý logic "Submit" (Khóa form, chuyển trạng thái).

#### Bước 3: Module Kết nối ASP (Tuần 3)
1.  Tạo API Key cho ASP System.
2.  Viết API `GET` danh sách hồ sơ (Tối ưu query JSON).
3.  Viết API `POST` nhận kết quả xét duyệt.
4.  *Test giả lập:* Dùng Postman đóng vai trò là ASP để gọi vào PHP Portal kiểm tra luồng duyệt.

#### Bước 4: Student Portal & Realtime Data (Tuần 4)
*Đây là phần khó nhất về mặt hạ tầng mạng.*

*   **Vấn đề:** PHP Portal (Internet) thường không thể gọi trực tiếp vào ASP (LAN nội bộ) để lấy điểm số realtime (Giai đoạn 3).
*   **Giải pháp đề xuất:**
    *   **Phương án 1 (Sync 1 chiều):** ASP định kỳ (hoặc khi có điểm mới) sẽ gọi API `POST /api/internal/sync-grades` lên PHP Portal để lưu vào database trung gian. PHP Portal chỉ đọc từ DB này. (Khuyên dùng - Ổn định nhất).
    *   **Phương án 2 (Tunneling):** Nếu bắt buộc realtime 100%, IT phải mở port hoặc dùng VPN/Tunnel để PHP gọi ngược về ASP.

#### Bước 5: Testing & Deployment
1.  UAT (User Acceptance Testing): Test luồng user nộp -> Admin ASP duyệt -> User nhận email.
2.  Triển khai lên Server (VPS/Hosting).
3.  Cài đặt SSL (HTTPS) để bảo mật thông tin thanh toán.

### V. ĐOẠN MÃ MẪU (LARAVEL CONTROLLER)

Dưới đây là ví dụ cho API quan trọng nhất: **ASP cập nhật trạng thái hồ sơ**.

```php
// ApplicationController.php

public function updateStatus(Request $request)
{
    // 1. Validate dữ liệu từ ASP gửi lên
    $validator = Validator::make($request->all(), [
        'application_id' => 'required|exists:applications,id',
        'status'         => 'required|in:approved,request_info,rejected',
        'student_code'   => 'required_if:status,approved',
        'reason'         => 'required_if:status,request_info',
    ]);

    if ($validator->fails()) {
        return response()->json(['error' => $validator->errors()], 422);
    }

    // 2. Tìm hồ sơ
    $app = Application::find($request->application_id);

    // 3. Xử lý logic theo trạng thái
    if ($request->status == 'approved') {
        $app->status = 'approved';
        $app->student_code = $request->student_code;
        // Logic gửi email Chúc mừng nhập học...
    } 
    elseif ($request->status == 'request_info') {
        $app->status = 'request_info';
        $app->review_note = $request->reason;
        // Logic gửi email Yêu cầu bổ sung...
    }

    $app->save();

    return response()->json(['message' => 'Cập nhật thành công'], 200);
}
```

### VI. LƯU Ý QUAN TRỌNG

1.  **Bảo mật API cho ASP:** Không để lộ endpoint `/api/internal/*` ra ngoài. Nên đặt IP Whitelist (chỉ IP tĩnh của công ty mới gọi được) hoặc dùng cơ chế Basic Auth kết hợp Token bí mật.
2.  **Ảnh chứng từ:** Khi ASP lấy JSON hồ sơ về, link ảnh phải là đường dẫn public (hoặc signed URL nếu dùng S3) để nhân viên trong mạng LAN có thể xem được ảnh mà không cần login vào portal.
3.  **Xử lý lỗi:** Cần log lại mọi request từ ASP để tra soát nếu có lỗi đồng bộ (Ví dụ: ASP báo duyệt rồi nhưng PHP bị lỗi mạng chưa update được).
